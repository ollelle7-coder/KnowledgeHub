## 주요 결정

- KH는 세부 프로젝트 관리를 모두 대체하지 않는다.
- KH의 핵심 역할은 Capture / Route / Surface / Track이다.
- KH Dashboard의 1차 핵심은 단순 모니터링이 아니라 명령 생성, 전달, 수행 상태 추적, 결과 회수, 표현이다.
- 각 AI 도구 화면을 그대로 임베드하는 방식보다, 명령과 결과를 프로젝트 단위로 추적하는 방식을 우선한다.
- KH Dashboard는 Flutter가 아니라 React / Next.js 기반 웹 콘솔로 생각한다.
- KH DB는 지식 원본 저장소가 아니라 운영 상태 저장소로 둔다.
- 초기 DB는 SQLite가 적합하다.
- KS는 당장 DB화하지 않고 Markdown / Git 원본을 유지한다.
- KS DB화는 나중에 색인 DB / 검색 인덱스 / Vector DB 방식으로 검토한다.
