# KH Overview - 2026-04-29

## 1. 정의

KnowledgeHub(KH)는 여러 프로젝트를 세밀하게 대신 관리하는 일반 PM 도구가 아니다.

KH는 운영자의 순간적인 생각, 프로젝트 상태, 판단 필요 항목, 도구/에이전트 실행 흐름을 한 곳에 모아 보여주는 의사결정 보조 관제 시스템이다.

## 2. 현재 전제

현재 메인 관리 대상 프로젝트는 다음 네 개다.

- EOB
- Cafe_ops
- KS / KnowledgeStorage
- Warship

KS는 KH의 상위 개념이 아니라 KH가 관제하는 프로젝트 중 하나다. 다만 지식 저장소라는 성격 때문에 다른 프로젝트의 확정 결과물이 KS로 승격될 수 있는 특수성을 가진다.

## 3. KH의 핵심 역할

KH의 핵심 역할은 네 가지다.

```text
1. Capture
   운영자의 생각, 메모, 외부 입력을 놓치지 않게 받는다.

2. Route
   수집된 입력을 프로젝트, 작업, 문서, 판단 항목으로 보낸다.

3. Surface
   지금 운영자가 봐야 할 판단거리와 막힌 지점을 위로 올린다.

4. Track
   열려 있는 작업과 최소 상태만 추적한다.
```

## 4. KH가 하지 말아야 할 것

KH는 모든 프로젝트의 세부 진행을 강제로 표준화하지 않는다.

하지 말아야 할 것:

- 모든 프로젝트에 동일한 PM 항목 강제 적용
- 진행률, 마일스톤, 협업자, 릴리즈, 운영, 리스크를 무조건 공통화
- 프로젝트별 문서와 판단 내용을 KH 내부에 중복 저장
- KS의 역할을 KH가 대체
- 허브 자체가 또 하나의 과도한 관리 대상이 되는 것

## 5. 기본 구조

```text
KH
├─ Hub Setup
├─ Operator Inbox
├─ Project Creation Pipeline
├─ Project Registry
└─ Project Profiles
```

## 6. 허브 독립 항목

허브 독립 항목은 KH 자체를 설계하고 운영하기 위한 영역이다.

```text
Hub Setup
- KH 목적 정의
- 대시보드 정보 구조
- 권한/승인 규칙
- 도구/에이전트 연결 방식
- 운영 원칙

Operator Inbox
- 빠른 메모
- 아이디어 수집
- 음성/텍스트 입력
- 미분류 보관
- 프로젝트/업무 라우팅 후보

Project Creation Pipeline
- 새 프로젝트 생성 기준
- 초기 셋업 절차
- 기본 문서/프로필 생성
- 프로젝트 타입별 추가 셋업
- 필요한 스킬/에이전트 연결
```

## 7. 프로젝트 공통 항목

공통 항목은 최소화한다.

```text
Project Common 최소 항목
- 프로젝트명
- 현재 상태
- 다음 판단 필요 항목
- 열려 있는 작업
- 막힌 지점
- 최근 중요 변화
- 관련 링크
```

이 이상은 프로젝트 특화 항목으로 보낸다.

## 8. 프로젝트 특화 항목

프로젝트별 상세 관리는 각 프로젝트 프로필 안에 둔다.

예시:

```text
EOB
- 세계관
- 캐릭터
- 에피소드 플롯
- Save the Cat 구조
- 설정 충돌

Cafe_ops
- 앱 기능 정의
- UX / 화면
- 카페 운영 요구사항
- 재고/발주/직원관리
- 개발 태스크

KS
- 문서 구조
- 정보 승격 기준
- 확정 지식 관리
- 스킬 패키징
- stale 문서 점검
- 승격 후보 관리

Warship
- 게임 콘셉트
- 함선 설계
- 전투 시스템
- 밸런스
- 콘텐츠 구조
```

## 9. Project Creation Pipeline 정의

Project Creation Pipeline은 프로젝트의 일상 관리가 아니라, 아이디어를 관리 가능한 프로젝트로 승격시키는 초기 셋업 절차다.

```text
새 프로젝트 아이디어
↓
프로젝트 생성 여부 판단
↓
기본 정보 입력
↓
프로젝트 타입 선택
↓
기본 산출물 생성
↓
타입별 추가 셋업
↓
필요 스킬/에이전트 연결
↓
대시보드 등록
```

## 10. Operator Inbox 정의

Operator Inbox는 운영자의 기억력 한계를 보완하기 위한 수집 장치다.

목적은 정리 자체가 아니라 휘발 방지다.

```text
스팟 아이디어 / 메모 / 생각
↓
빠른 입력
↓
미분류 보관
↓
라우팅 검토
↓
프로젝트 / 작업 / 문서 / 결정 후보로 연결
```

## 11. KH와 KS의 관계

```text
KH = 판단과 운영을 돕는 관제 허브
KS = 확정 지식을 보존하는 프로젝트/저장소
```

정확한 관계:

```text
틀린 표현:
Hub가 KS 승격 후보를 관리한다.

정확한 표현:
Hub는 KS 프로젝트 안에 있는 승격 후보 관리 항목을 관제판에 올려준다.
```

## 12. 현재 판단

KH는 독립 프로젝트로 분리하는 것이 맞다.

이유:

- KS와 KH는 역할이 다르다.
- KS는 지식 저장소 설계/운영 프로젝트다.
- KH는 여러 프로젝트를 관제하는 운영 허브다.
- GPT 프로젝트 그룹도 분리하는 것이 컨텍스트 오염을 줄인다.
